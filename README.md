# receipt-split-manager

レシートの商品項目を家計簿用データとして整理するためのWebアプリです。高額品は購入日の実支出として記録しつつ、家計簿上の入力予定を数ヶ月に分けて作成・管理できます。

初期版は手動入力を中心にしつつ、ブラウザ内OCRによるレシート画像の読み取りにも対応しています。電子レシート取り込みや画像の自動結合はまだ実装していません。

## 主な機能

- 商品入力
  - 購入日
  - 店舗名
  - レシート上の商品名
  - 正式な商品名
  - 金額（税込）
  - 分類
  - 通常入力 / 分割入力
- 分割入力
  - 分割月数
  - 開始月
  - 月額入力額の自動計算
  - 端数は最終月で調整
  - メモ
- 今月の入力予定
  - 今月入力すべき分割項目を表示
  - 未入力合計と予定合計を表示
- 月別予定一覧
  - 月ごとの配分入力予定を表示
  - 入力済み / 未入力の切り替え
- 登録済み商品一覧
  - 実際に購入した商品の記録を表示
- 学習辞書
  - 過去に入力したレシート表記、正式名、分類、店舗名、金額帯、確定回数、最終使用日を保存
  - 同じレシート表記を入力したときに候補を表示
- レシート読取
  - カメラ撮影、画像選択、複数画像選択
  - OCR結果をテキスト欄へ反映
  - 読み取り結果の手修正
  - 1行ごとの商品候補化
- データ管理
  - JSONバックアップ / 復元
  - 商品一覧CSV / 分割予定CSVの出力

## データの考え方

このアプリでは、以下を別データとして扱います。

- 実支出: 実際に購入日に支払った金額
- 配分入力: 家計簿上で月ごとに割り当てる金額

家計簿上の入力額は税込金額を基準にしています。

## データ保存

データはブラウザの `localStorage` に保存されます。

外部サーバー、ログイン、クラウド同期は使用していません。個人情報や入力内容は外部送信されません。

`localStorage` 保存のため、以下の場合にデータが消える可能性があります。

- ブラウザを変更した場合
- 別の端末で開いた場合
- ブラウザのキャッシュやサイトデータを削除した場合
- プライベートブラウズなど保存が残らない環境で使った場合

## 起動方法

依存関係をインストールします。

```bash
npm install
```

開発サーバーを起動します。

```bash
npm run dev
```

ブラウザで以下を開きます。

```text
http://localhost:5173/
```

スマホ実機から確認する場合は、同じWi-Fi内のPCでホストを指定して起動します。

```bash
npm run dev -- --host 0.0.0.0
```

PCのローカルIPアドレスを確認し、スマホのブラウザで以下のようにアクセスします。

```text
http://PCのIPアドレス:5173/
```

例:

```text
http://192.168.1.10:5173/
```

## スマホ実機でのOCR確認

1. `npm run dev -- --host 0.0.0.0` で起動します。
2. スマホをPCと同じWi-Fiに接続し、`http://PCのIPアドレス:5173/` を開きます。
3. 「レシート読取」タブを開きます。
4. 「カメラで撮影」を押し、スマホ標準カメラでレシートを撮影できるか確認します。
5. 「画像を選択」を押し、保存済み画像を1枚選択できるか確認します。
6. 「複数画像を選択」を押し、複数画像を選択できるか確認します。
7. OCR結果が「OCR結果の貼り付け・修正」欄へ入るか確認します。
8. 必要に応じてテキストを手修正し、「1行ごとに分解」で候補化できるか確認します。

カメラ利用はブラウザ権限、端末、ブラウザ仕様、`HTTPS` / `localhost` 条件に依存します。スマホからPCのIPアドレスでアクセスする場合、ブラウザによってはカメラ起動やOCR関連の動作に制限が出ることがあります。

## ビルド方法

```bash
npm run build
```

ビルド結果は `dist/` に出力されます。
